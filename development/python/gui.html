    tkinter github link
    - native python gui, but sometimes need manually install tcl on linux
    - can be exported (at least for win, linux) to monofile executable use pyinstaller command pyinstaller myscript.py --onefile --noconsole
    - gui widgets look very different on different platforms
    - gui widgets can't display unicode symbols with position(number) greater then 65535. This situation can appear for example when you try to display text from your email, which you read use handmaded python script. As result will be reised error, then app crush. The solution for this case is check the string to wrong symbols, replace this to unicode code number or just delete, before displaying use gui widget.
    - gui widgets look ugly
    - default gui widgets haven't id parameters. You can extend class manually, but this make feel insane.
    - on different platforms some widgets can have little lags look displaying behavior. For example the last pressed button can look like freezed in pressed condition, but continue work as expected
    - tkinter have some gui builder tools/frameworks like pmw (link bottom)

    wxPython github link
    - can be build as monofile exe (at least windows, linux) use pyinstaller syntax pyinstaller myscript.py --onefile --noconsole
    - gui widgets look different on different platforms
    - widgets have different behavior on different platforms (reversed z-index on windows... next widget add to back, but linux add to front)
    - on linux support GTK3 themes as result colors look close to OS colors

    pyqt5 download link
    - work done, have lot of docs, tutorials, paper books
    - can't be easy builded as monofile exe use pyinstaller
    - need qt installed on system where you want run pyqt app (if you distribute your app, your target machine need qt installed)
    - pyqt5 and qt is two different libs. Qt is crossplatform gui tool. PyQt5 is tool which call qt from python side
    docs

    http://zetcode.com/gui/pyqt5/

    eel github link
    - look like Electron HTML/JS GUI apps, but result have less size. Electron "hello world" ~80mb, eel "hello world" ~7mb
    - can be build as monofile exe use pyinstaller sytnax pyinstaller myscript.py --onefile --noconsole
    - need installed chrome/chromium on target machine
    - for server side uses python microframework bottle github link
    - for render widgets uses web browser engine (chrome/chromium)
    docs

    https://github.com/ChrisKnott/Eel#usage
